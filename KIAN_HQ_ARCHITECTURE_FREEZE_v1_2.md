## Architecture Rules (DO NOT CHANGE)

-   Never create modules based on roles.
-   Never use if(role == "EXECUTIVE") as the primary architecture
    pattern.
-   All permissions MUST be handled by RBAC.
-   Roles are collections of permissions.
-   New features MUST follow existing module architecture.

3.  Never send unnecessary context.

## Development Rules for AI Agents

Before implementing any feature, ALWAYS ask: 1. Does this follow Module
Based Architecture? 2. Does this follow RBAC? 3. Is AI really required?
4. Can this be solved using business logic instead? 5. Is this feature
within project scope? 6. Is this scalable for future roles and modules?
7. Is it optimized for Cloudflare Free Tier? 8. Is it mobile friendly?
9. Is it secure by default? 10. Is token usage optimized?

If the answer to any question is NO, the implementation MUST be
reconsidered.
